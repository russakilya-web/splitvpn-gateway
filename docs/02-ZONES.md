# 📡 Зоны маршрутизации

## Три типа зон

VPN Gateway использует **policy-based routing** для разделения трафика на три зоны.

---

## 🔒 VPN Zone (192.168.1.20-30)

**Весь трафик идёт через VPN**

Идеально для:
- Устройств, где VPN нужен постоянно
- Smart TV для просмотра зарубежного контента
- Игровых консолей

### Как добавить устройство

Отредактируйте `/etc/dhcp/dhcpd.conf`:

```bash
host my-smart-tv {
    hardware ethernet AA:BB:CC:DD:EE:FF;  # MAC-адрес устройства
    fixed-address 192.168.1.25;            # IP в диапазоне 20-30
}
```

Перезапустите DHCP:
```bash
sudo systemctl restart isc-dhcp-server
```

---

## 🚀 Bypass Zone (192.168.1.31-49)

**Трафик идёт напрямую, минуя VPN**

Идеально для:
- Устройств, которым VPN мешает (банковские приложения)
- Локальных сервисов
- Высокоскоростных загрузок

### Как добавить устройство

```bash
host my-bank-phone {
    hardware ethernet BB:CC:DD:EE:FF:00;
    fixed-address 192.168.1.35;            # IP в диапазоне 31-49
}
```

---

## ⚡ Split Zone (все остальные)

**Только указанные домены через VPN**

Это режим по умолчанию для всех устройств с IP вне VPN и Bypass зон.

Работает так:
1. Устройство делает DNS-запрос (например, youtube.com)
2. dnsmasq проверяет список доменов
3. Если домен в списке — IP добавляется в ipset
4. iptables маркирует пакеты к этим IP
5. Маркированные пакеты идут через VPN

### Управление доменами

**Через веб-интерфейс** (рекомендуется):
- Откройте http://192.168.1.1:8080
- Добавляйте/удаляйте домены

**Вручную:**
```bash
# Редактировать файл доменов
sudo nano /opt/vpn-gateway/vpn-domains.txt

# Применить изменения
sudo /usr/local/sbin/update-domains.sh
```

## Как работает routing

```
┌─────────────────────────────────────────────────────────────┐
│                         Входящий пакет                        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  IP Rule Priority 10: src 192.168.1.20-30 → table vpn       │
│  (VPN Zone - весь трафик через VPN)                          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  IP Rule Priority 20: src 192.168.1.31-49 → table bypass    │
│  (Bypass Zone - весь трафик напрямую)                        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  IP Rule Priority 100: fwmark 0x1 → table vpn                │
│  (Split Zone - пакеты к IP из ipset vpn_dst)                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  IP Rule Priority 110: table main suppress_prefixlength 0    │
│  (Fallback - обычный интернет через WAN)                     │
└─────────────────────────────────────────────────────────────┘
```

## Проверка зон

```bash
# Текущие правила маршрутизации
ip rule show

# Таблица vpn
ip route show table vpn

# Таблица bypass
ip route show table bypass

# Просмотр ipset (для Split зоны)
sudo ipset list vpn_dst | head -20
```
